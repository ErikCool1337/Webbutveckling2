  1. Varför används storyboard?: En enkel prototyp av webbplattsen med fokus på funktionalitet istället för grafik. Man kan kalla storyboards för webbplatsens riktning.
  2. Varför används wireframe?: wireframe är en grov skiss eller struktur av en webbplatts
  3. ? Vilken metod kan användas när en klient frågar efter en resurs?: GET-metoden är när en klient skickar en GET förfrågan. Servern svarar med den begärda resursen om den finns tillgänglig.
    POST-metoden: Denna metod används för att skicka data till servern för att behandlas och lagras. Det kan vara användbart när klienten vill skicka in data, till exempel när man fyller i ett webbformulär.
  4. Vad är det som är skillnad mellan HEAD och GET metod?:
    HEAD - Samma som get men får bara svaret som innehåller http huvudet
    GET: servern svarar med den begärda resursen om den finns tillgänglig.
  5. Pa vilket sätt genomförs “Hallway testning”? Slumpmässiga individer används för att testa webbplatsen.
  6. grafisk designer -> "look and feel". FrontEnd programmerare -> HTML- och CSS-dokument. Användarupplevelsedesigner -> Storyboards. Backend-programmerare -> PHP-skript.
  7. Vad gör W3C? World Wide Web Consortium, är en internationell organisation som ansvarar för att utveckla och fastställa webbstandarder och riktlinjer. De arbetar med att definiera och främja öppna webbstandarder som HTML, CSS och JavaScript. W3C spelar en viktig roll i att säkerställa att webben förblir öppen, interoperabel och tillgänglig för alla användare oavsett vilken webbläsare eller enhet de använder.
  8. html -> Identifierar text som en rubrik på andra nivån (h2). CSS -> Gör alla rubriker på andra nivån (h2) blå. JavaScript -> Kontrollerar ett formulärfält för en giltig inmatning. Ruby -> Skapar en anpassad webbapplikation på serversidan
  9. Frontend webbutveckling fokuserar på det som användaren ser och interagerar med på webbplatsen. Frontend-utvecklare arbetar med HTML, CSS och JavaScript för att skapa användargränssnittet, layouten och interaktionerna. De fokuserar på att skapa en bra användarupplevelse.
    Backend webbutveckling är ansvarig för den "bakom kulisserna" delen av webbplatsen. Backend-utvecklare arbetar med servern, databaser och serverapplikationer. De hanterar datalagring, hantering av användaruppgifter, och att säkerställa att webbapplikationen fungerar korrekt och säkert. Backend-utvecklare fokuserar på att göra webbplatsen funktionell och hantera data och logik.

Ett FTP-verktyg (File Transfer Protocol) används för att överföra filer mellan din lokala dator och en fjärrserver, vanligtvis för att publicera en webbplats eller överföra filer till en webbserver. Med ett FTP-verktyg kan du enkelt ladda upp och hämta filer från servern. För att få ett FTP-verktyg kan du ladda ner och installera det från internet. Det finns många gratis och kommersiella FTP-klientprogram tillgängliga för olika operativsystem. Några exempel på populära FTP-klienter inkluderar FileZilla, Cyberduck och WinSCP.

  3. "Responsive webbdesign" innebär att skapa en flexibel webbplats som automatiskt anpassar sig och ser bra ut på olika skärmstorlekar och enheter.
1d. Uppgifter
  1. Varför navänds Switch och Router? Switch och router används inom nätverk för olika ändamål: Switch: En switch används för att ansluta flera enheter i ett lokalt nätverk (LAN) och effektivt rikta data endast till rätt mottagare för snabbare kommunikation inom nätverket. Router: En router kopplar samman olika nätverk och styr dataflödet mellan dem genom att avgöra den bästa vägen för data att resa och hantera IP-adressering.
  2. Varför används internet exchange points? Internet exchange points är knutpunkter som internetleverantörer använder för att ansluta sig till internetets infrastrukturstomme
  3. Vilken roll har en protokoll i webbens sammanhang? I webbens sammanhang är ett protokoll en uppsättning regler och överenskommelser som definierar hur data ska överföras och tolkas mellan webbserver och webbläsare. Exempel inkluderar HTTP (Hypertext Transfer Protocol) som används för att hämta webbsidor och HTTPS (HTTP Secure) för säker kommunikation.
  4. Att Internet inte har något "single point of failure" innebär att det inte finns en enda central punkt eller enhet som om den skulle misslyckas skulle få hela Internet att krascha. Internet är decentraliserat och består av en mängd nätverk och serverar över hela världen, vilket gör det motståndskraftigt mot större störningar.
  5. Datorer och enheter utväxlar information genom att dela data i små paket som skickas över nätverket. Dessa paket innehåller information om avsändare, mottagare och själva datan. Paketen ruttas sedan genom nätverket till mottagaren.
  6. På vilket sätt kan vägen som denna information passerar spåras? Tracerroute
  7. Från vilka delar består ett domännamn? Domännamn är IP-adresser men istället för en massa nummer så blir det istället ett namn som vi männsikor förstår och kan tolka. Läskt till höger av domännamnet så finns top-level domains (TDL)
